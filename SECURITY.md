# Security Policy

## Supported Versions

Security fixes are made on the active `develop` branch and promoted to `main` through the normal release flow.

Published npm releases should be upgraded to the latest stable version unless a maintainer explicitly documents an exception.

## Reporting a Vulnerability

Please do not open a public issue for suspected vulnerabilities.

Report security concerns privately to the maintainer or through GitHub private vulnerability reporting if it is enabled for this repository.

Include:

- affected version or commit;
- Homebridge, Node.js, and operating system versions;
- steps to reproduce;
- expected impact;
- logs or proof-of-concept details that are safe to share.

The maintainer will acknowledge the report, assess severity, and coordinate remediation and disclosure.
